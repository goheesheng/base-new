# Base Batches — Past Cohorts (seed dataset)

This is a **seed** file. The first version is intentionally thin — fill it as you research past cohorts. Treat as a working notebook, not gospel. When in doubt, link to the announcement post and quote it directly.

## Why this exists

Selectors look for patterns; you should too. Knowing what past cohorts looked like:
- helps the user pick a track,
- shows what kinds of Base primitives win,
- surfaces gaps (under-represented categories) where a strong submission stands out.

## How to fill it

For each cohort:
1. Find the official announcement (Base/Coinbase blog, announcement tweet, Demo Day recap).
2. List teams in the order they appeared in the post.
3. For each team, record: track, one-liner, Base primitive used (if knowable), team links, public demo link if any.
4. If you can't verify a fact, leave it blank — don't guess.

## Patterns to watch for

When you have enough data, fill in:

- **Most-used Base primitives**: e.g., Smart Wallet, Mini Apps, OnchainKit, ...
- **Most-represented categories**: DeFi / consumer / infra / agents / identity ...
- **Under-represented categories** (opportunity zones for new applicants).
- **Common winning archetypes**: e.g., "Mini App with a clear distribution loop in Farcaster."
- **Common losing archetypes**: e.g., "Generic L2 protocol fork with no Base-specific surface."
